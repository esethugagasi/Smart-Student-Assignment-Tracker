Smart Student Assignment Tracker 

System Specification

1.	Introduction

1.1.	Project Title
Smart Student Assignment Tracker

1.2.	Domain
Education Technology

1.3.	Domain description
The education domain includes systems that support teaching, learning, and academic administration. Students frequently handle many classes and must keep track of assignments, projects, and deadlines. By allowing for centralized task management, digital solutions can assist organize academic tasks and increase productivity.

1.4.	Problem statement
Students are struggling to coordinate assignments across numerous modules. Deadlines might be missed due to poor organization or a lack of awareness of future obligations. Existing methods are frequently fragmented or not purposefully intended for academic task management.

The suggested solution solves this challenge by offering a single platform for students to track assignments, manage modules, and receive notifications about upcoming deadlines.

1.5.	Individual Scope
The scope of this project is limited to a system for students to organize academic assignments and track their progress. The system will enable basic user authentication, module management, assignment tracking, and dashboard reporting.
The project is feasible for a single developer because the system focuses solely on key functions and avoids extensive integration or support.
 
2.	System Overview
The Smart Student Assignment Tracker is a web-based application that allows students to:
•	Create an account and login.
•	Create and manage academic modules.
•	Create assignments related to modules.
•	Establish timelines and priorities.
•	Track assignment progress.
•	View upcoming deadlines on a dashboard.
•	Receive reminder alerts.

This approach is intended to assist students improve their time management skills and better arrange their academic workload more effectively.

3.	System Users
Students are the primary users of the system.
Responsibilities include:
•	creating and managing modules
•	Adding Assignments
•	Update assignment status.
•	Monitoring upcoming deadlines
•	Tracking progress using the dashboard
 

4.	Functional Requirements
The system must support the following functionalities:

User Management

•	The system must allow students to create accounts.
•	The system should allow students to log in securely.
•	Students must be able to log out of the system.

Module Management

•	The system must enable students to create modules.
•	The system must enable students to edit module details.
•	The system should allow students to delete modules.

Assignment Management

•	The system must enable students to create assignments.
•	The system must allow students to establish deadlines for assignments.
•	The system must allow students to set priority tasks.
•	The system must allow students to update assignment status.

Dashboard

•	The system must display upcoming assignments.
•	The system must show overdue assignments.
•	The system must display assignment progress.

Notification System

•	The system should notify students about upcoming deadlines.
•	Notifications can be provided via email or in-app alerts.
 

5.	Non-functional Requirements

Usability
•	The system's interface should be straightforward and user-friendly.

Performance
•	The system should respond to user input.

Security
•	User authentication must be enabled to protect student data.

Reliability
•	The system should keep accurate assignment data and prevent data loss.

Maintainability
•	The system should be constructed with modular components to allow for future growth.

6.	Assumptions
•	Users will have Internet access.
•	Students will be responsible for handling their own assignments.
•	Only student users will be supported in the first edition of the system.

7.	Constraints
•	The system was created by a single developer.
•	Advanced features, such as AI scheduling, are excluded.
•	The system will first be deployed as a web-based application.

8.	Feasibility
The proposed solution is suitable for a single project because it employs common web technologies and well-known software design principles. The functionalities are limited to basic assignment tracking functions, so the system can be completed within the semester's timetable.
