Stakeholder Analysis – Smart Student Assignment Tracker


Stakeholder - Student
Role - Primary user who manages assignments and deadlines
Key Concerns - Easy tracking, reminders and simple UI
Pain Points - Missed deadlines, poor planning and scattered tools
Success Metrics - ≥90% of assignments submitted on time

Stakeholder - Lecturer
Role - Assigns coursework and monitors submissions
Key Concerns - Students submitting on time and clear tracking of work
Pain Points - Late submissions and lack of visibility
Success Metrics - 10% reduction on late submissions

Stakeholder - System Administrator
Role - Manages system deployment and maintenance
Key Concerns - System uptime, reliability and performance
Pain Points - Downtime, difficult maintenance
Success Metrics - ≥99% uptime

Stakeholder - Developer
Role - Designs and maintains system
Key Concerns - Clean architecture and scalability
Pain Points - Complex code structure and hard debugging
Success Metrics - Faster feature delivery and reduced bugs

Stakeholder - University Administrator
Role - Monitors academic performance
Key Concerns - Improved student outcomes
Pain Points - Lack of centralized tracking
Success Metrics - Improved pass rates

Stakeholder - IT Support Staff
Role - Handles user/system issues
Key Concerns - Stability and easy troubleshooting
Pain Points - Frequent issues and unclear logs
Success Metrics - Reduced support tickets by 40%
