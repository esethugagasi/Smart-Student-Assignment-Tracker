# Reflection – Assignment 5

This assignment focused on translating system requirements into use case models and test cases. One of the main challenges was ensuring consistency with previous assignments. The use cases had to align directly with the functional requirements defined earlier, which required careful mapping and validation.

Another challenge was simplifying complex system behavior into clear use cases. For example, assignment management includes multiple actions such as creating, editing, and deleting tasks. These had to be grouped logically while still maintaining clarity.

Balancing detail and simplicity was also difficult. Use case specifications needed to be detailed enough to describe system behavior but not overly complex. This required focusing on the main success scenarios while including relevant alternative flows.

Developing test cases required thinking from a validation perspective rather than a design perspective. Each test case had to clearly map to a requirement and define measurable expected outcomes. This ensured traceability and improved system reliability.

Non-functional testing introduced another level of complexity. Performance and security requirements are harder to test compared to functional features. Defining realistic test scenarios such as concurrent users and login failures helped address this challenge.

Overall, this assignment improved understanding of how requirements are transformed into system behavior and validation processes. It highlighted the importance of traceability, consistency, and clarity in software engineering. The experience also demonstrated how structured documentation supports system development and testing.

This process ensures that the final system will meet stakeholder expectations while maintaining quality and reliability.
